TRENDLINES — FEATURES LIST
===========================

Current version: v1.2
Repo: https://github.com/KobiLDN/Trendlines

This file tracks what the site currently does, and what's planned next.
Bump the version in js/version.js whenever you commit a meaningful change.


SHIPPED (v1.2)
--------------
[x] Multiple trendlines per chart, color-coded:
      - Green = support (connect swing lows)
      - Red   = resistance (connect swing highs)
    Draw as many of each as you like; all lines stay on the chart.
[x] Undo Last button (removes the most recent trendline or cancels
    an in-progress draw).
[x] Clear All button (removes every trendline).
[x] Version badge pinned to the top-right of every page's nav bar,
    linking to FEATURES.txt changelog.

SHIPPED (v1.1)
--------------
[x] Version footer + version constant in js/version.js

SHIPPED (v1.0)
--------------
[x] Learn page with 5 core swing-trading concepts in plain English
    (trendline, action line, safety line, entry, risk:reward)
[x] Chart page using TradingView Lightweight Charts
[x] Click-to-draw trendline (two anchor points)
[x] Auto-suggests today's action line price from drawn trendline
[x] Visual action / safety / target lines on the chart
[x] Live risk:reward ratio (color-coded: green >=2, amber 1-2, red <1)
[x] Position-size calculator from account size + risk %
[x] Watchlist with localStorage persistence
[x] Status badge per setup: armed / triggered / stopped
[x] Export-to-alerts.json button on watchlist
[x] GitHub Action: daily platinum data refresh (yfinance)
[x] GitHub Action: 30-min alert checker, Telegram notifications
[x] Initial 2 years of platinum daily data committed

VERSIONING CONVENTION
---------------------
v1.0    Initial release

v1.x    Bump on every user-visible change
v2.0    Reserved for major change (e.g. backend rewrite, framework swap,
        or breaking storage migration)

The version string lives in: js/version.js
It appears as a badge at the top-right of every page automatically,
plus a small footer link.
